# UX/UI Audit — Account, auth, onboarding & alerts

## Scope

Routes: `app/account/**`, `app/auth/**`, `app/onboarding`, `app/dashboard`, `app/alerts/**`, `app/fee-alerts/**`, `app/rate-alerts/**`, `app/plans/**`, `app/pricing`, `app/billing-policy`, `app/shortlist/**`, `app/portfolio`, `app/my-briefs`, `app/my-learning`, `app/unsubscribe/**`, `app/newsletter/**`

Components: `components/ClaimAnonymousOnAuth.tsx`, `components/BookmarkButton.tsx`, all auth/login/signup form components under `app/auth/`.

## Healthy patterns

- **Password visibility toggles** — all three auth forms implement proper `<button type="button">` with distinct `aria-label` text ("Show/Hide password"), not `<span>` or decorative divs. Good.
- **`autocomplete` attributes** — `email`, `current-password`, and `new-password` values are set consistently on all auth inputs. This is the single most impactful password-manager hint and it is correctly in place.
- **Submit-button loading states** — every form disables its submit button and replaces text with a present-progressive label ("Sending…", "Creating account…", "Signing in…") during async operations. No double-submit risk.
- **Open-redirect guard** — `app/auth/callback/route.ts:103–106` validates the `next` param is a same-origin relative path before redirecting. Correctly prevents open redirects.
- **Anonymous-save claim idempotency** — `ClaimAnonymousOnAuth` uses a sessionStorage stamp per `user.id` and server-side upserts, so repeated tab navigations do not result in duplicate claims.
- **Auth error page** — `app/auth/auth-error/page.tsx` provides genuinely actionable, reason-specific copy for each failure mode (`otp_expired`, `pkce_failed`, `missing_params`) rather than a generic error, with an inline resend form. Well designed.
- **`role="switch"` pattern** — `DigestToggle.tsx` and `WatchlistAlertsToggle.tsx` both implement the correct `role="switch" aria-checked` pattern with focus rings.
- **`noValidate` + custom validation** — auth forms use `noValidate` to suppress native browser validation bubbles and provide custom, consistent error messages instead.
- **`aria-pressed` on toggle buttons** — `RateAlertSignupForm.tsx` uses `aria-pressed` correctly for the product/frequency toggle buttons (stateless toggle, not on/off switches).
- **`<details>` for technical error info** — `app/auth/auth-error/page.tsx:83–91` hides raw Supabase error text under a `<details>` summary so it is available for support but doesn't clutter the user-facing message.
