# R2 Audit — Loading / empty / error states

## Scope

Audited: `/app/compare`, `/app/advisors`, `/app/articles`, `/app/rates`, `/app/search`, `/app/quiz`, `/app/get-matched`, `/app/find-advisor`, `/app/account` (dashboard, dashboard sub-routes, bookmarks, watchlist, goals, holdings, notifications, quizzes, decisions, net-worth, saved-searches, timeline), `/app/community`, `/app/broker/[slug]`, `/app/shortlist`, `/app/switch`, `/app/best`, `/app/contact`, `/app/advisor-apply`. Also sampled: `components/Skeletons.tsx`, `components/directory/EmptyState.tsx`, `components/RouteErrorBoundary.tsx`.

## Quick wins

1. **`app/account/dashboard/loading.tsx`** — copy the `DashboardSkeleton` import from `app/account/loading.tsx`; one file, ~3 lines. Immediate improvement for the highest-traffic logged-in page.
2. **`<Suspense><CompareNav /></Suspense>`** — add `fallback={<div className="h-11 animate-pulse bg-slate-100 rounded-lg" />}` in `app/compare/page.tsx:153`. One-line fix, eliminates nav pop-in.
3. **Three missing `error.tsx` files** — `best/`, `switch/`, `benchmark/` each need a single file that re-exports `RouteErrorBoundary`. Copy-paste pattern from `app/advisor/error.tsx` (1 line each).
4. **Community route coverage** — `app/community/error.tsx` and `app/community/loading.tsx` both missing; a single copy each covers the entire community section via Next.js inheritance.
5. **Articles empty state** — replace the bare `<p>` with an `EmptyState` + "Clear search" button; the component is already imported and used throughout the codebase.

---

## Notable

- **Strong coverage on the core directories**: `/compare`, `/advisors`, `/rates` all have proper `loading.tsx`, `error.tsx`, and Suspense usage with meaningful skeletons that match the real layout. The `EmptyState` component and `RouteErrorBoundary` shared primitives are well-designed and consistently used where they have been adopted.
- **Pattern gap in account sub-routes**: The top-level `/account` route has `loading.tsx` + `error.tsx`, but 30+ sub-routes (`/account/dashboard`, `/account/decisions`, `/account/quizzes`, `/account/timeline`, `/account/net-worth`, etc.) have **neither**. Since each sub-route is `force-dynamic` and fetches from Supabase, they all rely on the parent skeleton, which does not match the sub-page layout and may not trigger if the user navigates within the account section.
- **Form states are well-handled**: `/find-advisor`, `/advisor-apply`, `/get-matched`, `/quiz`, `/community/new`, and `/account/profile` all track `submitting`/`isPending` states, surface inline errors, and redirect on success. No missing submit-pending states found in sampled forms.
- **`Suspense fallback={null}` is used intentionally** for low-priority strips (`NextActions`, `PushNotificationOptIn`) where invisible loading is acceptable. The `BrokerReviewClient` use is the one case that warrants a skeleton since it wraps the primary page content.
