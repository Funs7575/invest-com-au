# UX/UI Audit — Brokers & investing directory

## Scope

Routes: `app/brokers` (308 → /compare), `app/compare`, `app/compare/[versus]`, `app/versus`, `app/broker/[slug]`, `app/best`, `app/best/[slug]`, `app/costs/[slug]`

Components: `BrokerCard.tsx`, `BrokerComparisonTable.tsx`, `BrokerStickyRightRail.tsx`, `BrokerReliabilityScore.tsx`, `BrokerLogo.tsx`, `ArticleBrokerTable.tsx`, `ComparisonTableSkeleton.tsx`, `CompareCrossSellBanner.tsx`, and all `components/directory/*`

| 1 | High | a11y | `components/BrokerStickyRightRail.tsx:46–49` | `aria-hidden={!visible}` hides the aside from AT when invisible but `pointer-events-none` does not remove the elements from the tab order. When `visible=false`, keyboard users still tab into the "Dismiss" button and the affiliate CTA, reaching invisible interactive elements. | Add `inert` (or `tabIndex={-1}` on each child button/link when `!visible`) so keyboard focus cannot reach off-screen content. | 2 | High | a11y | `components/BrokerComparisonTable.tsx:97–114` | Desktop comparison table `<th>` headers have no `scope` attribute. | 3 | High | a11y | `components/BrokerComparisonTable.tsx:66–68` | Sort control uses `role="tablist"` + `role="tab"` for what is purely a sort-button group, not a tab panel relationship. Screen readers announce the buttons as tabs (implying a panel will change) when they actually sort a single visible table. | Remove `role="tablist"` / `role="tab"`. ## Healthy patterns

- **`CompareSelectionBar`** implements `style={{ paddingBottom: "max(0rem, env(safe-area-inset-bottom))" }}` for safe-area handling on iOS — correct.
- **`components/directory/*` primitives** are used consistently: `SearchInput`, `FilterChips`, `FilterPill`/`FilterPopover`, `EmptyState`, and `ResultCount` are all wired in `CompareClient`. The brokers domain shows the highest adoption of the directory primitive system relative to other audited domains.
- **`BrokerLogo`** correctly implements `priority` prop for LCP-candidate logos and has a graceful `onError → LetterFallback` path with animated blur placeholder.
- **`CompareDesktopTable`** implements `aria-sort` on sortable column headers and `scope="col"` — good practice.
- **`BrokerReliabilityScore`** sets `aria-expanded` on the toggle button and uses a proper `<form>` with labelled radio inputs.
- **ISR revalidation** is set on all key routes (`/compare`: 1800s, `/broker/[slug]`: 3600s, `/best/[slug]`: 3600s, `/compare/[versus]`: 3600s) — consistent and appropriate for data freshness.
- **`CompareCrossSellBanner`** uses `role="complementary"` + `aria-label` and is dismissible via `sessionStorage` — clean UX and good ARIA usage.
- **Live region** at `CompareClient.tsx:811` (`aria-live="polite" aria-atomic="true"`) announces filter result count changes to AT — well implemented.
- **`ComparisonTableSkeleton`** provides a detailed structural skeleton that closely mirrors the real table, reducing perceived jank on slower connections.
