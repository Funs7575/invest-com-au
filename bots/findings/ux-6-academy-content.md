# UX/UI Audit — Academy, courses & editorial content

## Scope

Routes audited: `app/academy`, `app/article/[slug]`, `app/articles`, `app/articles/[category]`, `app/how-to/[slug]`, `app/glossary`, `app/glossary/[term]`, `app/questions`, `app/questions/[slug]`, `app/topic/[slug]`, `app/tag/[slug]`, `app/courses`, `app/courses/[slug]`, `app/my-learning`, `app/authors/[slug]`, `app/reviewers/[slug]`.

Components audited: `ArticleCover.tsx`, `ArticleSidebar.tsx`, `ArticleKeyTakeaways.tsx`, `ArticleCategoryFilter.tsx`, `ArticleSearchInput.tsx`, `ArticleComments.tsx`, `ArticleBrokerTable.tsx`, `AuthorByline.tsx`, `AskQuestionForm.tsx`, `CollapsibleSection.tsx`, `OnThisPage.tsx`.

## Healthy patterns

- **`ArticleCover` is a proper unified component.** Uses `next/image` with `fill`, correct `sizes`, `priority` prop for LCP, and a graceful gradient-placeholder fallback — no broken images on uncovered articles.
- **`LastUpdatedBadge` emits a `<time datetime="...">` element** with machine-readable ISO dates and a human-readable `aria-label`. Used consistently on `/article/[slug]`.
- **`ArticleKeyTakeaways` is `aria-label`'d and SVG bullets are `aria-hidden`.** The section ID wired to speakable schema is correct GEO practice.
- **Loading skeletons exist for all major editorial routes** (`app/articles/loading.tsx`, `app/article/[slug]/loading.tsx`, `app/glossary/loading.tsx`, `app/courses/[slug]/loading.tsx`) — perceived performance is well handled.
- **Empty-state copy is contextual** — the articles hub distinguishes between "no search results" and "no category results" with different suggestion text.
- **`AskQuestionForm` uses proper `<label htmlFor>` + `id` pairing** for all its fields (lines 99, 113, 125) via `sr-only` labels, and has a `role="alert"` error container with `aria-describedby`.
- **Comment submission feedback uses `role="status"` for success and `role="alert"` for errors** (`ArticleComments.tsx:208,216`), which correctly differentiates polite vs assertive live regions for screen readers.
- **Article section IDs use `scroll-mt-24`** to prevent the sticky header from covering the target heading when navigating via the in-page TOC.
- **`AuthorByline` changelog entries use `formatDate` with `{ style: "long" }`** and the reviewer line is a proper `<p>` with a `<Link>` — good credibility signal with accessible link text.
