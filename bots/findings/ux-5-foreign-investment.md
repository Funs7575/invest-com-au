## Healthy patterns

- **Banner token discipline**: `components/foreign-investment/banner-tokens.ts` is well-structured, referenced correctly by `IntentCountryBadge.tsx`, `IntentCountryRecommendationCard.tsx`, and `CountryRuleAlerts`. The three-layer banner stack (pill → rule alerts → recommendation) follows documented ordering.
- **Country intent lifecycle**: `RememberCountry.tsx` + `ClearIntentCountryButton.tsx` + `getIntentCountry()` server read form a clean cookie-based intent loop with no client/server state mismatch.
- **DirectoryBanners usage on main directories**: `app/invest/page.tsx`, `app/compare/page.tsx`, `app/advisors/page.tsx` all correctly use `<DirectoryBanners surface=...>` rather than assembling the stack manually.
- **`lib/page-recommendations.ts` single source of truth**: Recommendation copy is correctly externalised; no hardcoded recommendation strings found in the component layer.
- **Form a11y on calculators**: `<label htmlFor>` correctly associates every form control in `FirbFeeEstimatorClient.tsx` and `NonResidentDividendClient.tsx`. `aria-label` on the DTA search input is present.
- **`overflow-x-auto` on all data tables**: Tax tables, DTA tables, and savings-rate comparison tables all wrap in `overflow-x-auto` containers — mobile table overflow is handled.
- **Compliance disclaimers via single source**: `FOREIGN_INVESTOR_GENERAL_DISCLAIMER`, `DTA_DISCLAIMER`, `DASP_WARNING`, `FIRB_DISCLAIMER` are all sourced from `lib/compliance.ts`, not hardcoded, across the entire domain.
- **`CountryHubTemplate` config-driven architecture**: Adding a new country hub requires only a config object in `lib/foreign-investment-country-data.ts` and a four-line route file. Zero JSX duplication across 12 country pages.
- **ExpatPlanClient progressbar aria**: `role="progressbar"` with `aria-valuenow`/`aria-valuemin`/`aria-valuemax` and a contextual `aria-label` is correctly implemented in `ExpatPlanClient.tsx:99–103`.
