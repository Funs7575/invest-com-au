# UX/UI Audit — Calculators & financial tools

## Scope

All routes matching `app/*-calculator` (compound-interest, cgt, dasp, debt, dividend-reinvestment, fire, franking-credits, investment-income-tax, mortgage, non-resident-dividend, portfolio, property-vs-shares, property-yield, retirement, savings, smsf, super-contributions, switching, tco, trade-cost, us-share-costs) plus `app/calculators`, `app/tools`, `app/fee-simulator`, `app/fee-impact`, `app/fee-tracker`, `app/tax-optimizer`, `app/portfolio-calculator`, `app/portfolio-xray`, `app/scenario`, `app/scenarios`, `app/research-tools`, `app/quick-audit`, `app/health-scores`.

Components inspected: `CalculatorShell.tsx`, `CalculatorExplainButton.tsx`, `CalculatorHistory.tsx`, `CalculatorLeadCapture.tsx`, `CalculatorShareButton.tsx`, `CalcShared.tsx` (shared sub-components for the hub).

**Calculators that do NOT use `CalculatorShell`:** Every standalone `app/*-calculator` page builds its own full-page layout. `CalculatorShell` is only used by `RdTaxCalculator` (embedded on grant pages, not a standalone `/calculator` route). The standalone calculators instead use two distinct layout families:
- **Hero-gradient + full-page flow** (retirement, mortgage, savings, smsf): large violet/rose hero, inputs card, explicit "Calculate" CTA button, results shown below on click.
- **Two-column split** (compound-interest, fire, dividend-reinvestment): inputs panel left, live-updating results panel right, no submit button.
- **Hub inline widgets** (`app/calculators`): `CalcSection` wrapper from `CalcShared.tsx`, shared across ~25 mini-calculators embedded in the hub.

## Healthy patterns

- **`InvestmentIncomeTaxClient.tsx`** is the gold standard for calculator accessibility: uses `useId()` for label–input association, `inputMode="decimal"`, `aria-describedby` for hints, `role="region" aria-live="polite" aria-atomic="true"` on the results pane, and `aria-label="Breadcrumb"` on the nav. New calculators should follow this template.

- **`CgtCalculator.tsx` (hub embed)** has an excellent empty/initial state: a dashed-border card with an icon and "Enter a Capital Gain" — avoids showing meaningless $0 results by default.

- **`CalculatorShareButton.tsx`** uses `aria-label="Copy shareable link to this calculation"` and a proper `<svg aria-hidden>` pattern — a good model for all icon-only or ambiguous action buttons.

- **`RelatedCalculators` component** (`components/RelatedCalculators.tsx`) exists and is used by SMSF and compound-interest — a consistent cross-linking pattern awaiting wider adoption.

- **`non-resident-dividend-calculator`** slider uses `htmlFor`/`id` association correctly (`NonResidentDividendClient.tsx:185,192`) — one of the few range inputs in the domain that is fully accessible.

- **`ComplianceFooter variant="calculator"`** is consistently applied on the page-level `page.tsx` files for all standalone `*-calculator` routes; the compliance contract is respected for the core calculator estate.

- **URL state sync** (`useUrlSync` in `CalcShared.tsx`, `useCalculatorState` hook) enables deep-linkable, shareable results across most calculators — a strong UX differentiator.
