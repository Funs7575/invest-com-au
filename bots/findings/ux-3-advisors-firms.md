# UX/UI Audit — Advisors & firms

## Scope

Routes audited: `app/advisors`, `app/advisor/[slug]`, `app/advisor-apply`, `app/advisor-guides`, `app/advisor-jobs`, `app/advisor-portal`, `app/advisor-signup`, `app/find-advisor`, `app/find`, `app/firm/[slug]`, `app/pros`, `app/teams`, `app/booking/success`, `app/consultations`, `app/concierge`, `app/expert`, `app/family-office`.

Components audited: `AdvisorDirectory.tsx`, `AdvisorCompareMatrix.tsx`, `AdvisorMatchCTA.tsx`, `AdvisorReviewForm.tsx`, `AdvisorTrustScoreCard.tsx`, `AdvisorVideoIntro.tsx`, `AdvisorPhotoUpload.tsx`, `AdvisorOptInCheckboxes.tsx`, `AdvisorAppointmentsWidget.tsx`, `AdvisorCalendarEmbed.tsx`, `BookingWidget.tsx`, `ConciergeBookingHandoff.tsx`.

| 1 | **Critical** | a11y — label association | `components/AdvisorReviewForm.tsx:159,195,209,223` | All four `<label>` elements for "Did you use services", "Your name", "Title", and "Your review" use `className="block …"` but have **no `htmlFor` or `id`** on the paired `<input>`/`<textarea>`. The `<span id={labelId}>` approach is used in `StarRatingInput` but the four standalone labels below are plain `<label>` elements without `htmlFor`. | Add a unique `id` to each input/textarea and a matching `htmlFor` on each `<label>` — or wrap each input inside its `<label>`. |

## Healthy patterns

- **`AdvisorReviewForm`** correctly uses `role="radiogroup"` + `aria-labelledby` for star-rating groups, `aria-checked` on radio buttons, and `role="alert"` for validation errors — well above average.
- **`AdvisorAppointmentsWidget`** is a proper `<form onSubmit={…}>` with labelled inputs (`htmlFor`/`id`), `role="alert"` on errors, and a concurrency-safe slot-claim flow with user-friendly conflict messaging.
- **`find-advisor/page.tsx` ProgressBar** uses `role="progressbar"` + `aria-valuenow/min/max/label` correctly (from `components/ui/ProgressBar.tsx`).
- **`AdvisorOptInCheckboxes`** provides meaningful collapse toggle with `aria-expanded` + `aria-label`, and a proper `<label>` wrapping each checkbox option.
- **Directory page (`AdvisorsClient.tsx`)** correctly imports all shared directory primitives (`SearchInput`, `SortDropdown`, `TabBar`, `FilterPanel`, etc.) rather than reimplementing them.
- **`ConciergeBookingHandoff`** is a clean, focused component with `data-testid` attributes, proper `<Link>` usage, and no side effects.
- **Server-side rendering** is used correctly throughout — route pages (`page.tsx`) are RSC while interactive sections are isolated to `"use client"` leaf components.
