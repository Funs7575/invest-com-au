# R2 Audit — SEO & Metadata

## Scope

Second-pass audit focussed on SEO signals, metadata completeness, structured data, and crawlability.
Examined ~700 `page.tsx` / `layout.tsx` files in `app/`, cross-referenced with `lib/seo.ts`,
`lib/schema-markup.ts`, `app/sitemap.ts`, and `app/robots.ts`.

Lens: page routes missing `generateMetadata`/`metadata` export, thin/duplicated titles, missing
`alternates.canonical`, missing OG + Twitter images, JSON-LD coverage, breadcrumb consistency,
sitemap coverage, robots/noindex correctness, and heading-H1 uniqueness.

## Quick Wins

1. **Community thread metadata** (`app/community/[category]/[threadId]/page.tsx:130`): add `openGraph`, `twitter`, `alternates.canonical` — 4-line addition to `generateMetadata` return block. Immediate social-share improvement for all UGC content.
2. **Topic pages in sitemap** (`app/sitemap.ts`): 13 known slugs, static list — one PR to add them to shard 0 static array.
3. **`best-for/[slug]` twitter card** (`app/best-for/[slug]/page.tsx:67`): single line addition `twitter: { card: "summary_large_image" as const }`.
4. **Double-branded titles**: strip `"| Invest.com.au"` suffix from the ~52 static title strings. The layout template appends the brand automatically; the suffix is redundant.
5. **`howToJsonLd` import fix** (`app/how-to/[slug]/page.tsx:11`): switch import to `lib/schema-markup.ts` to get dynamic `datePublished`/`dateModified` instead of the hardcoded 2025-01-15 stub.

## Notable

- **Root layout baseline is solid**: `metadataBase`, global `openGraph`, `twitter: summary_large_image`, and `robots: { index: true }` are all correctly set in `app/layout.tsx:56–96`. Pages that rely on the default OG image (`/api/og`) get a usable share card; the issues are additive, not structural.
- **Broker review pages (`/broker/[slug]`) and article pages (`/article/[slug]`) are fully correct**: dynamic OG image via `/api/og/broker`, twitter card, canonical, breadcrumb, and multi-schema JSON-LD (`FinancialProduct` + `Article` + `BreadcrumbList` + conditional `FAQPage`). These are the template to copy.
- **Glossary pages have excellent JSON-LD** (`DefinedTerm` + `QAPage` + `BreadcrumbList`) but the OG/twitter gap means the GEO signal is not reinforced in social previews.
- **`howToJsonLd` duplication**: the function exists in both `lib/seo.ts` (used by how-to pages, legacy signature) and `lib/schema-markup.ts` (richer, used by `app/switch/[type]`). The `lib/schema-markup.ts` comment at line 874 acknowledges this. Consolidate to avoid schema drift.
- **Absolute vs relative canonicals**: 329 pages use `SITE_URL` prefix for canonicals; ~180 use relative paths. Both are valid given `metadataBase`, but the mix adds risk ahead of the Oct–Dec 2026 domain migration. Relative paths (`/path`) are safer.
