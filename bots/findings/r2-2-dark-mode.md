# R2 Audit — Dark mode

## Is dark mode shipped?

**YES — dark mode is fully implemented and shipped.**

- `ThemeProvider` (`components/ThemeProvider.tsx`) adds/removes the `dark` class on `<html>` using `localStorage` preference or `prefers-color-scheme`.
- A flash-of-wrong-theme inline script in `app/layout.tsx:173` runs before React hydrates, reading `localStorage.theme` and toggling `html.dark` so there is no flash.
- A `ThemeToggle` button (`components/ThemeToggle.tsx`) exposes a three-way `light / dark / system` cycle and is rendered in the navigation header.
- `app/globals.css` has ~200 lines of `html.dark .classname { … !important }` overrides that blanket-remap the most common Tailwind utility classes to dark-surface tokens (`--bg-card: #1e293b`, body `#0f172a`, etc.).

**However, coverage is uneven.** The global CSS override approach means dark mode "mostly works" for standard Tailwind classes, but there are several categories of gaps documented below.

## Scope

Search was performed across `app/` and `components/` (excluding `.next/` and `node_modules/`).

- **Total `dark:` variant usages in source:** 41 lines across ~6 components — extremely sparse for a ~1 000-file codebase. Nearly all dark-mode work is done by the global CSS override rules, not inline `dark:` Tailwind variants.
- **Files containing `bg-white`:** ~100+ distinct source files. All are theoretically covered by `html.dark .bg-white { background-color: var(--bg-card) !important }`.
- **Files using inline `style={{ background/color: "#hex" }}`:** 14 non-API source files. These are **not** covered by the class-name-based global overrides.
- **Custom color families (`text-ink-*`, `bg-coral-*`):** ~39 usages across 4 files — no corresponding `html.dark .text-ink-*` overrides exist.
- **Opacity-white variant gaps (`bg-white/15`, `/25`, `/30`, `/40`, `/70`):** ~90 usages across 20+ files — only `/5`, `/10`, `/20`, `/50`, `/60`, `/80`, `/90`, `/95` are in the global override list.

## Quick wins

1. **Add missing opacity-white variants to globals.css** — four one-liners cover `bg-white/15`, `/25`, `/30`, `/40`, `/70`. Fixes 90+ usages across 20 files instantly.
2. **Add `html.dark .text-ink-800` and `html.dark .text-ink-900` rules** — fixes invisible card titles on `InvestListingCard` and `HomeActivitySection` (the main marketplace listing components).
3. **Add `html.dark .bg-white\/70` rule** — fixes mortgage calculator summary cards and account goals progress tracks.
4. **Wrap `RateChangesToday` with Tailwind classes** — it's a small component (~90 lines); replacing the outer `style={{ background: "#fff" }}` with `className="bg-white dark:bg-slate-800"` fixes the white-blob issue on every page that embeds it.

## Notable

- **Specificity trap (finding #1):** The `!important` global overrides silently defeat the handful of authored `dark:bg-*` classes on shared components (`Navigation`, `FilterPanel`, `FilterPill`, `MobileBottomNav`, `CalculatorShell`). The dark mode still looks correct in practice because the global `--bg-card: #1e293b` is visually equivalent to what `dark:bg-slate-800` would produce — but the discrepancy between authored intent and rendered output will cause confusion and bugs as the codebase evolves. The long-term fix is to remove `!important` from all global dark-override rules (or scope them with `:where()` to lower specificity) so `dark:` variants can override when present.

- **Three entire authenticated-user pages** (`my-briefs`, `my-learning`, `teams/referrals`) were written entirely in inline React styles with hardcoded light-palette hex values. These are post-login screens where dark-mode users are likely to be — they would look severely broken in dark mode. Estimated refactor cost: 2–3 hours each.

- **The global CSS override approach works reasonably** for the public-facing marketing and listing pages, which use standard Tailwind classes throughout. The gaps are concentrated in (a) post-login authenticated pages written with inline styles, (b) two custom color families (`ink-*`, `coral-*`) used in the marketplace listing card and home feed, and (c) opacity-white tint variants used in calculators and account pages.
