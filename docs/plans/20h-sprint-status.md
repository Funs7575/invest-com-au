## Cycle protocol (each fire)
1. **Pre-flight:** `git fetch origin main`; confirm main `ci.yml` is green; read this
   doc + the two queues.
2. **Discover:** rotate — a bot pass (`/ai-journey` vs Netlify, real-user personas)
   and/or `/audit-remediation-scout`. Verify every finding with retries (the proxy
   throws transient 403/503) → append to the relevant queue.
3. **Build / fix:** drain the queue — build-heavy from `BUILD-EVERYTHING-QUEUE`,
   fixes from `REMEDIATION_QUEUE`. One mergeable chunk, or up to 3 via parallel
   worktree agents.
4. **Verify (local):** `tsc --noEmit <changed>` + `vitest run <changed/related>` +
   `eslint <changed>`. Red → fix or drop. Never merge a red iteration.
5. **Merge:** open PR; when local-green **and** `ci.yml` green → merge via API per
   tier (A/B immediately, C after a 30-min quiet window). Ignore the 3 infra reds.
6. **Verify (bots):** next cycle, re-run the touched journey vs the freshly-deployed
   Netlify mirror to confirm the fix landed.
7. **Log:** append a dated line to the Live log below. The auto-revert workflow
   backstops any bad merge (~5 min).

## Stop conditions
Queue drained (`LOOP_DONE` sentinel) · usage floor reached · an avoid-list / Tier-E
item needs the founder · founder says stop.

## Durability note
A self-paced in-session `/loop` runs as long as this session lives. For guaranteed
survival across a disconnect, the same protocol can run as a cloud `/schedule` (cron,
survives terminal close) — state lives here, so either driver resumes identically.
