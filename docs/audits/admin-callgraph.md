## ✅ Expected / acceptable callers

### `app/api/cron/**` (75 files)
Batch jobs running server-side on a schedule with no user session. Service-role
is the correct client — these jobs write to tables the anon/session key cannot
touch (e.g. cron heartbeats, bulk email stamps, audit triggers). No refactor
needed unless a cron also handles user-provided input (none identified).

### `app/admin/**` (29 files) and `app/api/admin/**` (40 files)
Admin dashboard pages and their API routes. Protected by the `requireAdmin`
gate in `proxy.ts` (JWT `app_metadata.role = "admin"` check). Service-role
is correct here — admin operators need to write to any table. No refactor
needed.
